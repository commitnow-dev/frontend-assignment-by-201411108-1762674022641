### 👀 과제의 설명

## 개요

React를 사용해서 인스타그램의 피드와 관련된 기능을 구현합니다.

별도의 전역 상태 관리 라이브러리를 사용하지 않고 React의 기능만을 활용해서 개발합니다.

API가 별도로 제공되지 않기 때문에 브라우저 storage를 사용하여 기능을 개발합니다.

그 외 별도 라이브러리 제약은 없습니다.

​

### 기능 요구사항

\[화면 구성]

화면은 아래와 같은 3가지 화면으로 구성하세요.

* **피드 보기** 화면 업로드 된 피드를 확인할 수 있는 화면입니다.
* **피드 업로드** 사진을 선택해서 간단한 내용과 해시태그를 담아 피드를 생성할 수 있는 화면입니다.
* **내 피드 모아보기** 내가 업로드한 피드를 확인할 수 있는 화면입니다. 각 피드를 누르면 피드 보기 화면과 동일한 UI가 제공됩니다.

​

\[세부 기능]

1. 피드는 아래 인터페이스를 참고하여 개발하세요. 필요하다면 인터페이스를 직접 추가할 수 있습니다.

```typescript
interface Creator {
  id: string;
  name: string;
}

interface Feed {
  id: string;
  content: string;
	imageUrls: string[];
	hashtags?: string[];
	creator: Creator;
	createdAt: Date;
}
```

3. 피드 업로드 시에는 다음과 같은 제약 조건이 있습니다. 제약 사항을 하나라도 준수하지 못할 경우 피드를 생성할 수 없도록 만들어야 합니다. 피드 업로드 시 스토리지에 저장하여 피드 보기에서 활용하세요.&#x20;

   * 토스트와 같은 안내 메시지를 추가하거나 버튼 상태에 제약 사항과 관련된 내용을 반영하세요.
   * 반드시 한 장 이상의 이미지가 포함 되어야 합니다. 이미지는 jpg, png, webp 파일만 사용할 수 있습니다.
   * 내용이 반드시 포함 되어 있어야 합니다.
   * 해시태그는 비어 있을 수 있습니다.
4. 피드에서는 업로드한 이미지를 가로 슬라이드를 통해서 넘겨 볼 수 있습니다. 피드의 디자인은 인스타그램을 클론하여 개발하세요.&#x20;
5. 내 피드 모아보기는 피드의 첫 번째 이미지를 대표 이미지로 grid 형태로 구성하세요.&#x20;
6. 피드를 누를 경우 “피드 보기”에서 개발한 컴포넌트를 재사용하여 해당 피드의 내용을 확인할 수 있어야 합니다.&#x20;


### 🎯 과제의 요구사항

- 전역 상태 관리와 관련된 라이브러리 없이 React의 기능만 활용하세요.
- 브라우저 스토리지를 서버 대신 사용하세요.
- 피드 업로드 개발을 진행할 때는 form을 활용하세요.
- form과 관련된 제약 조건을 정확하게 구현하세요.
- 재사용성을 고려하여 컴포넌트를 개발하세요.
